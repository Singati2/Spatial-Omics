Prerequisites: Core Concepts You Need First

What Is an Estimator?

An estimator is a recipe (formula) that takes data and produces a guess for some unknown quantity.

In our research: the unknown quantity is the true CCI score $T_{lr}^{\text{true}}$ (how strongly ligand $l$ and receptor $r$ interact in the tissue). We cannot observe this directly because we only measured some genes. Our estimator $\hat{T}_{lr}$ uses imputed (reconstructed) gene expression data to guess $T_{lr}^{\text{true}}$.

The naive estimator: Compute CCI directly from imputed data, ignoring the fact that imputation introduced errors.

$$\hat{T}_{lr}^{\text{naive}} = \frac{1}{|\mathcal{E}|}\sum_{(i,j) \in \mathcal{E}} \hat{\lambda}_{il} \cdot \hat{\lambda}_{jr}$$

The corrected estimator: Subtract the estimated bias (our regression calibration).

$$\hat{T}_{lr}^{\text{corrected}} = \hat{T}_{lr}^{\text{naive}} - \hat{\rho}_{lr}\sqrt{\widehat{\text{MSE}}(l) \cdot \widehat{\text{MSE}}(r)}$$
Analogy — A Bathroom Scale: Imagine your bathroom scale consistently reads 2 kg too high. Your true weight is the unknown. The scale reading is the naive estimator. Subtracting 2 kg is the corrected estimator. The "2 kg" is the bias. But the scale also fluctuates randomly by $\pm 0.3$ kg each time — that's the variance. Our goal: find an estimator with small bias AND small variance.
What Is a Simulation with Known Ground Truth?

In real data, we never know the true CCI score — that's why we need estimators. But in a simulation, WE create the data, so WE know the truth. This lets us:

  1. Generate fake cells with known expression rates $\boldsymbol{\Lambda}$ (we choose these)
  2. Simulate Poisson counts from these rates
  3. Pretend we only measured some genes (panel selection)
  4. Run our imputation + CCI pipeline
  5. Compare our estimates $\hat{T}_{lr}$ to the truth $T_{lr}^{\text{true}}$ (which we know because we created the data)
  6. Repeat 1000+ times to see how the estimator behaves on average

Bias of the Corrected Estimator Under Simulation

1.1 What Is Bias? (The Simplest Possible Explanation)

Definition of Bias

The bias of an estimator $\hat{\theta}$ for a true parameter $\theta$ is:

$$\text{Bias}(\hat{\theta}) = \mathbb{E}[\hat{\theta}] - \theta$$

In words: bias = average estimate minus truth.

  • If $\text{Bias} > 0$: the estimator overestimates on average
  • If $\text{Bias} = 0$: the estimator is unbiased (correct on average)
  • If $\text{Bias} < 0$: the estimator underestimates on average
Why "On Average" Matters

A single estimate can be off in any direction — that's random error. Bias is about the systematic direction. If you weigh yourself 1000 times on a biased scale, the average of those 1000 readings will be off by exactly the bias. Individual readings will scatter around that wrong average.

An unbiased estimator might give you 48, 53, 47, 51, 52, ... for a true value of 50. The average is ~50 (correct). A biased estimator gives 53, 58, 52, 56, 57, ... The average is ~55 (systematically 5 too high).

1.2 Bias in Our Specific Problem

Recall from the bias derivation: when we compute CCI on imputed data, the expected value of the naive estimator is:

$$\mathbb{E}[\hat{T}_{lr}^{\text{naive}}] = T_{lr}^{\text{true}} + \underbrace{\rho_{lr}\sqrt{\text{MSE}(l) \cdot \text{MSE}(r)}}_{\text{bias} > 0}$$

So:

$$\text{Bias}(\hat{T}_{lr}^{\text{naive}}) = \rho_{lr}\sqrt{\text{MSE}(l) \cdot \text{MSE}(r)} > 0$$

The naive estimator is positively biased — it makes interactions look stronger than they really are.

1.3 Bias of the Corrected Estimator

Our corrected estimator subtracts an estimate of the bias:

$$\hat{T}_{lr}^{\text{corrected}} = \hat{T}_{lr}^{\text{naive}} - \hat{b}_{lr}$$

where $\hat{b}_{lr} = \hat{\rho}_{lr}\sqrt{\widehat{\text{MSE}}(l) \cdot \widehat{\text{MSE}}(r)}$ is our bias estimate.

Bias of the Corrected Estimator
$$\text{Bias}(\hat{T}_{lr}^{\text{corrected}}) = \mathbb{E}[\hat{T}_{lr}^{\text{corrected}}] - T_{lr}^{\text{true}}$$ $$= \underbrace{\mathbb{E}[\hat{T}_{lr}^{\text{naive}}] - T_{lr}^{\text{true}}}_{\text{original bias } b_{lr}} - \underbrace{\mathbb{E}[\hat{b}_{lr}]}_{\text{estimated bias}}$$ $$= b_{lr} - \mathbb{E}[\hat{b}_{lr}]$$

In words: the corrected estimator's bias equals the original bias minus the average of our bias estimate.

  • If $\mathbb{E}[\hat{b}_{lr}] = b_{lr}$ (perfect bias estimation): corrected bias = 0. Ideal case.
  • If $\mathbb{E}[\hat{b}_{lr}] < b_{lr}$ (underestimate bias): corrected bias > 0. Under-correction — some false positives remain.
  • If $\mathbb{E}[\hat{b}_{lr}] > b_{lr}$ (overestimate bias): corrected bias < 0. Over-correction — may lose true interactions.

1.4 How to Measure Bias in Simulation

Step 1: Set up the ground truth.

Choose a true rate matrix $\boldsymbol{\Lambda}$ (e.g., $N = 5000$ cells, $G = 500$ genes, $K = 10$ programs). Compute the true CCI scores $T_{lr}^{\text{true}}$ for all LR pairs using the full, noise-free $\boldsymbol{\Lambda}$.

Step 2: Simulate one experiment.

Generate counts $y_{ig} \sim \text{Poisson}(\lambda_{ig})$. Select panel $S$ (300 genes). Impute the missing 200 genes. Compute $\hat{T}_{lr}^{\text{naive}}$ and $\hat{T}_{lr}^{\text{corrected}}$.

Step 3: Repeat $R$ times (e.g., $R = 1000$).

Each repetition $r$ gives different Poisson draws, so different counts, different imputations, different CCI estimates. Collect $\{\hat{T}_{lr}^{(1)}, \hat{T}_{lr}^{(2)}, \dots, \hat{T}_{lr}^{(R)}\}$ for both naive and corrected.

Step 4: Compute empirical bias. $$\widehat{\text{Bias}}_{\text{naive}} = \frac{1}{R}\sum_{r=1}^R \hat{T}_{lr}^{\text{naive},(r)} - T_{lr}^{\text{true}}$$ $$\widehat{\text{Bias}}_{\text{corrected}} = \frac{1}{R}\sum_{r=1}^R \hat{T}_{lr}^{\text{corrected},(r)} - T_{lr}^{\text{true}}$$

1.6 When Is Residual Bias Acceptable?

Relative Bias

To judge whether bias matters, compare it to the standard deviation:

$$\text{Relative Bias} = \frac{|\text{Bias}(\hat{T})|}{\text{Std}(\hat{T})}$$
  • Relative bias $< 0.1$: bias is negligible (dominated by random variation)
  • Relative bias $0.1 - 0.5$: bias is present but may be acceptable
  • Relative bias $> 0.5$: bias is the dominant source of error — must correct
Example 1.2 — Relative Bias Assessment

Continuing from Example 1.1:

  • Naive: $|\text{Bias}| = 0.64$, $\text{Std} = 0.42$, Relative Bias $= 0.64/0.42 = 1.52$ — bias dominates
  • Corrected: $|\text{Bias}| = 0.08$, $\text{Std} = 0.48$, Relative Bias $= 0.08/0.48 = 0.17$ — bias is minor

The naive estimator's error is primarily systematic (bias). The corrected estimator's error is primarily random (variance). This is exactly what we want — random error averages out with more data; systematic error does not.

Variance and Mean Squared Error Comparison

2.1 What Is Variance? (From Absolute Basics)

Variance Explained

The variance measures how much an estimator fluctuates across repeated experiments.

$$\text{Var}(\hat{\theta}) = \mathbb{E}\left[(\hat{\theta} - \mathbb{E}[\hat{\theta}])^2\right]$$

In words: variance = average squared distance from the estimator's own average.

Standard deviation $= \sqrt{\text{Var}}$ — has the same units as the estimator, easier to interpret.

Analogy — Archery:
  • Low bias, low variance: arrows clustered tightly around the bullseye. Ideal.
  • Low bias, high variance: arrows scattered widely but centered on the bullseye. Unbiased but imprecise.
  • High bias, low variance: arrows clustered tightly but away from the bullseye. Precise but wrong.
  • High bias, high variance: arrows scattered widely and off-center. Worst case.

2.2 Why Correction Can Increase Variance

Our corrected estimator is:

$$\hat{T}^{\text{corr}} = \hat{T}^{\text{naive}} - \hat{b}$$

The variance of the corrected estimator:

Variance of the Corrected Estimator
$$\text{Var}(\hat{T}^{\text{corr}}) = \text{Var}(\hat{T}^{\text{naive}}) + \text{Var}(\hat{b}) - 2\,\text{Cov}(\hat{T}^{\text{naive}}, \hat{b})$$

Key insight: if the bias estimate $\hat{b}$ is noisy (high $\text{Var}(\hat{b})$), the correction adds variance. This is the bias-variance tradeoff — reducing bias can increase variance.

2.3 Mean Squared Error: The Complete Picture

The MSE Decomposition (Most Important Formula in Statistics)
$$\boxed{\text{MSE}(\hat{\theta}) = \text{Bias}^2(\hat{\theta}) + \text{Var}(\hat{\theta})}$$

In words: total error = systematic error + random error.

MSE answers the question: "On average, how far is my estimate from the truth?" It accounts for BOTH bias and variance simultaneously.

Why MSE, not just bias or variance alone?

  • An estimator with zero bias but enormous variance is useless (individual estimates wildly wrong)
  • An estimator with zero variance but large bias is also useless (consistently wrong by the same amount)
  • MSE captures both — the estimator with lowest MSE is "best" in a squared-error sense

2.4 How to Compute MSE in Simulation

Empirical MSE Computation

From $R$ simulation replicates:

$$\widehat{\text{MSE}} = \frac{1}{R}\sum_{r=1}^R \left(\hat{T}^{(r)} - T^{\text{true}}\right)^2$$

This can be decomposed as:

$$\widehat{\text{MSE}} = \underbrace{\left(\frac{1}{R}\sum_{r=1}^R \hat{T}^{(r)} - T^{\text{true}}\right)^2}_{\widehat{\text{Bias}}^2} + \underbrace{\frac{1}{R}\sum_{r=1}^R \left(\hat{T}^{(r)} - \frac{1}{R}\sum_{r'}\hat{T}^{(r')}\right)^2}_{\widehat{\text{Var}}}$$

2.6 The MSE Ratio: When Is Correction Worth It?

MSE Ratio Test
$$\text{MSE Ratio} = \frac{\text{MSE}(\hat{T}^{\text{corrected}})}{\text{MSE}(\hat{T}^{\text{naive}})}$$
  • Ratio $< 1$: correction improves things — use corrected estimator
  • Ratio $= 1$: no change — correction neither helps nor hurts
  • Ratio $> 1$: correction makes things worse — keep naive estimator

The correction is beneficial when the bias² saved exceeds the variance added:

$$\text{Correction helps when: } \text{Bias}_{\text{naive}}^2 - \text{Bias}_{\text{corr}}^2 > \text{Var}(\hat{b})$$

2.7 Efficiency

Relative Efficiency

The relative efficiency of estimator A compared to B:

$$\text{RE}(A, B) = \frac{\text{MSE}(B)}{\text{MSE}(A)}$$

RE $> 1$ means A is more efficient (lower MSE). In our example:

$$\text{RE}(\text{corrected, naive}) = \frac{0.586}{0.236} = 2.48$$

The corrected estimator is 2.48 times more efficient — you would need 2.48× as many samples with the naive estimator to achieve the same MSE as the corrected estimator.

Coverage Probability of Confidence Intervals

3.1 What Is a Confidence Interval? (The Correct Interpretation)

Confidence Interval Definition

A $100(1-\alpha)\%$ confidence interval for parameter $\theta$ is a random interval $[L, U]$ (computed from data) such that:

$$P(\theta \in [L, U]) = 1 - \alpha$$

For a 95% CI ($\alpha = 0.05$): if you repeated the experiment infinitely many times and computed a CI each time, 95% of those intervals would contain the true value.

Common Misconception

A 95% CI does NOT mean "there is a 95% probability that the true value is in this interval." The true value is fixed (not random). The interval is random (it changes with each experiment). The probability statement is about the procedure, not any single interval.

Analogy — Fishing Net: You throw a net (CI) toward a fish (true value). Before throwing, there's a 95% chance the net lands around the fish. After throwing, the fish is either caught or not — the 95% described the accuracy of your throwing technique, not the state of any particular throw.

3.2 CI for the CCI Statistic

From our panel perturbation stability analysis, we get $B$ estimates $\{\hat{T}^{(1)}, \dots, \hat{T}^{(B)}\}$. The percentile confidence interval:

Percentile Confidence Interval
$$\text{CI}_{95\%} = \left[\hat{T}^{(\lfloor 0.025 \cdot B \rfloor)}, \; \hat{T}^{(\lceil 0.975 \cdot B \rceil)}\right]$$

where $\hat{T}^{(k)}$ is the $k$-th smallest value (order statistic).

Alternatively, using the normal approximation:

$$\text{CI}_{95\%} = \hat{T} \pm 1.96 \cdot \widehat{\text{SE}}$$

where $\widehat{\text{SE}} = \text{Std}(\hat{T}^{(1)}, \dots, \hat{T}^{(B)})$.

3.3 What Is Coverage Probability?

Coverage Probability

The coverage probability is the fraction of simulated CIs that actually contain the true value:

$$\text{Coverage} = \frac{1}{R}\sum_{r=1}^R \mathbb{1}\left[T^{\text{true}} \in \text{CI}^{(r)}\right]$$

where $\text{CI}^{(r)}$ is the confidence interval from the $r$-th simulation.

  • For a nominal 95% CI: target coverage = 0.95
  • Coverage $< 0.95$: CIs are too narrow (anti-conservative, overconfident)
  • Coverage $> 0.95$: CIs are too wide (conservative, underconfident)

3.5 Formal Test of Coverage

Test Whether Coverage Equals the Nominal Level

Under the null $H_0$: true coverage $= 1 - \alpha = 0.95$, the number of "hits" (CIs containing the truth) follows $\text{Binomial}(R, 0.95)$.

The test statistic:

$$Z = \frac{\hat{p} - 0.95}{\sqrt{0.95 \times 0.05 / R}}$$

where $\hat{p}$ is the observed coverage proportion. Reject if $|Z| > 1.96$.

For our example (naive): $Z = (0.782 - 0.95)/\sqrt{0.95 \times 0.05 / 1000} = -0.168/0.00689 = -24.4$. Massively significant — coverage is far below nominal.

For corrected: $Z = (0.938 - 0.95)/\sqrt{0.0000475} = -0.012/0.00689 = -1.74$. Not significant at $\alpha = 0.05$ — coverage is statistically consistent with 95%.

Type I Error Rate and Power

4.1 Hypothesis Testing: The Basics

The Two Hypotheses

For each LR pair, we test:

  • $H_0$ (null): There is no spatial interaction between ligand $l$ and receptor $r$. The CCI score is no larger than expected by chance.
  • $H_1$ (alternative): There IS a spatial interaction. The CCI score is significantly larger than chance.

We compute a $p$-value and reject $H_0$ if $p < 0.05$ (declaring the interaction "significant").

Key Rates
$$\text{Type I Error Rate} = \alpha = P(\text{reject } H_0 \mid H_0 \text{ is true}) \quad \text{(target: } 0.05\text{)}$$ $$\text{Power} = 1 - \beta = P(\text{reject } H_0 \mid H_1 \text{ is true}) \quad \text{(want: as high as possible)}$$ $$\text{Type II Error Rate} = \beta = 1 - \text{Power} = P(\text{fail to reject } H_0 \mid H_1 \text{ is true})$$
Analogy — Criminal Trial:
  • $H_0$: defendant is innocent
  • $H_1$: defendant is guilty
  • Type I Error: convicting an innocent person (false positive) — the system tries hard to avoid this
  • Type II Error: acquitting a guilty person (false negative) — unfortunate but less "bad" than convicting an innocent
  • Power: probability of correctly convicting a guilty person

4.3 Why Imputation Bias Inflates Type I Error

The Core Problem in Our Research

When the CCI test statistic is biased upward, the $p$-values are too small under the null. Pairs with no real interaction get small $p$-values and are falsely declared significant.

If nominal $\alpha = 0.05$ but actual Type I Error is 0.25, then 25% of null interactions are falsely discovered instead of the expected 5%.

4.4 How to Measure Type I Error in Simulation

Type I Error Rate Estimation

Key idea: Simulate data where $H_0$ is TRUE (no spatial interaction), then check how often the test incorrectly rejects.

Procedure:

  1. Generate data with independent ligand and receptor expression (no spatial co-patterning). Specifically: assign expression values to cells independently of their spatial locations.
  2. Run the full pipeline (imputation → CCI → permutation test)
  3. Record whether $p < 0.05$
  4. Repeat $R$ times
$$\widehat{\alpha} = \frac{1}{R}\sum_{r=1}^R \mathbb{1}[p^{(r)} < 0.05]$$

Target: $\widehat{\alpha} \approx 0.05$. If $\widehat{\alpha} \gg 0.05$, the test has inflated Type I error.

4.5 How to Measure Power

Power Estimation

Key idea: Simulate data where $H_1$ is TRUE (real interaction), then check how often the test correctly rejects.

Procedure:

  1. Generate data with a real spatial CCI pattern of known strength $\delta$ (e.g., ligand-high cells are spatially adjacent to receptor-high cells)
  2. Run the full pipeline
  3. Record whether $p < 0.05$
  4. Repeat $R$ times
$$\widehat{\text{Power}} = \frac{1}{R}\sum_{r=1}^R \mathbb{1}[p^{(r)} < 0.05]$$

Compute for different effect sizes $\delta$ to get a power curve.

4.7 The Power-Type I Error Tradeoff (ROC-Like Analysis)

Size-Adjusted Power Comparison

Since the naive test has inflated Type I error, a fair comparison adjusts for this. Set the rejection threshold so both tests have exactly 5% Type I error, then compare power.

For the naive estimator, achieve $\alpha = 0.05$ by using threshold $p < 0.008$ (instead of $p < 0.05$).

Size-adjusted power at $\delta = 1.0$:

  • Naive (at $p < 0.008$): 0.52
  • Corrected (at $p < 0.05$): 0.67

After size-adjustment, the corrected estimator has HIGHER power. The naive estimator's apparent power advantage was entirely due to its inflated false positive rate.

5.4 How MSE Behaves: The Complete Picture

MSE Scaling Laws
$$\text{MSE}_{\text{naive}} = \underbrace{\rho_{lr}^2 \cdot C^2 \cdot \frac{K^2 \lambda_{\max}^2}{m^2}}_{\text{Bias}^2} + \underbrace{\frac{1}{N} \cdot g(m, \bar{\lambda}, K)}_{\text{Var}}$$ $$\text{MSE}_{\text{corrected}} = \underbrace{\text{small residual}}_{\text{Bias}^2 \approx 0} + \underbrace{\frac{1}{N} \cdot g(m, \bar{\lambda}, K) + \frac{\text{Var}(\hat{b})}{1}}_{\text{Var (slightly higher)}}$$

5.6 Asymptotic Behavior

What Happens as $N \to \infty$?

Naive estimator:

$$\text{MSE}_{\text{naive}} \xrightarrow{N \to \infty} \text{Bias}^2 = \rho_{lr}^2 \cdot \frac{C^2 K^2 \lambda_{\max}^2}{m^2}$$

The variance vanishes, but the bias remains. The naive estimator converges to the wrong value. It is inconsistent for the true CCI score (unless $\rho = 0$).

Corrected estimator:

$$\text{MSE}_{\text{corrected}} \xrightarrow{N \to \infty} \text{residual bias}^2 \approx 0$$

Both bias and variance vanish. The corrected estimator converges to the true value. It is consistent.

This Is the Most Important Result

No matter how much data you collect ($N \to \infty$), the naive CCI estimator never converges to the truth when applied to imputed spatial data. It converges to $T^{\text{true}} + \text{bias}$. Only the corrected estimator converges to $T^{\text{true}}$.

This is why the correction is not optional — it's necessary for valid science. Without it, more data just gives you more confidence in the wrong answer.
